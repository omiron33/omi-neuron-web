# Task 4P.1: Visual Baseline Audit

## Objective
Document the current visual baseline and identify pain points that make the experience feel "vanilla."

## Requirements
- Capture screenshots and short recordings across small/medium/large graphs.
- Identify density, legibility, and interaction affordance issues.
- Note where motion is missing or feels static.
- Record current FPS/memory at baseline node counts.

## Deliverables
- `docs/visualization/visual-baseline.md` with findings and screenshots.
- Baseline performance notes for normal/degraded/fallback modes.

## Acceptance Criteria
- [ ] Baseline doc lists at least 5 concrete visual issues.
- [ ] Includes snapshots for 50/200/500 node scenarios.
- [ ] Baseline FPS/memory observations are documented.
